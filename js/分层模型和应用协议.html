网络分层模型和应用协议

遇到复杂问题的时候，可以使用分层的思想把问题简单化

  1. 每层相对独立，只需解决自己的问题
  2. 每层无需考虑上层的交付，仅需把自己的结果交给下层即可
  3. 每层有多种方案可供选择，选择不同方案不会对上下层造成影响
  4. 每一层会在上一层的基础上，增加一些额外信息

五层网络模型

网络解决的问题：两个程序员如何交换数据

  • 五层模型
  1. 应用层 跟具体应用相关的消息格式 HTTP FTP DNS SMTP POP3
  2. 传输层 如何保证消息的可靠传递 TCP UDP
  3. 网络层 如何在互联网中找到对象 IP 路由器
  4. 数据链路层 如何在一个子网中找到对方 MAC 交换机
  5. 物理层 上面给我的东西如何用信号表示？ 光纤 双绞线 同轴电缆 集线器

URL

是一个固定的字符串
http://a.com:80/news/detail?id=1#t1
从网络中哪台计算机(domain): a.com
中的哪个程序(port):80
寻找哪个服务(path) : /news/detail 并注明了获取服务的具体细节(path)
以及要用什么样的协议通信(schema):http
?id=1 是query

  • query参数只能是标准的ASCII字符，就是只能有数字英文，一些英文标点不能出现中文的
    #t1是hash
  • 当协议http端口为80时，端口可以省略
  • 当协议是https端口为443时，端口可以省略
  • schema、domain、path是必填的，其他的根据具体要求填写

GET POST PUT DELETE

  • Content-Type 消息类型，用的什么格式下面就要用对应方式写

HTTP响应状态码

1 信息、服务器收到请求，需要请求者继续执行操作
2 成功，操作被成功接收并处理
3 重定向，需要进一步的操作以完成请求
4 客户端错误，请求包含语法错误或无法完成请求
5 服务器错误，服务器在请求处理的过程中发生了错误

常见状态码

  1. 200 一切正常，OK
  2. 301 资源已被永久重定向
  3. 302 资源已被临时重定向
